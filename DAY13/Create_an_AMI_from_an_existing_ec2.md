- AMI is a preconfigured template blueprint used to launch virtual servers known as EC2 instances 

- It contains the O.S ,application software,configuration settings, storage volumes
- Navigate to the aws amanagement console and search  for EC2 

- Navigate to instances and select the required instance

- It is considered best to stop the instance

- Go to actions then select images and template -> create images

- Configure the details that is : 
    
    **Name of the AMI**

    **The description**

    **Choose the enable/clear the reboot option** 

    **Confirm the volume settings**

    **Create Image**
            

- Once  followed this instructions start the instance and wait for the AMI status to be available
